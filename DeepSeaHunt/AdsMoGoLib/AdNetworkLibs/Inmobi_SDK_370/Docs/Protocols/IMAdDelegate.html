IMAdDelegate Protocol Reference

Conforms to NSObject
Declared in IMAdDelegate.h

Overview

This is the delegate for receiving state change messages from an IMAdView. Use this to receive callbacks for banner ad request succeeding, failing or for the events after the banner ad is clicked.

Tasks

  • – adViewDidFinishRequest:

    Callback sent when an ad request loaded an ad. This is a good opportunity to add this view to the hierarchy if it has not yet been added.
  • – adView:didFailRequestWithError:

    Callback sent when an ad request failed. Normally this is because no network connection was available or no ads were available (i.e. no fill).
  • – adViewWillPresentScreen:

    Callback sent just before when the adView is presenting a full screen view to the user. Use this opportunity to stop animations and save the state of your application in case the user leaves while the full screen view is on screen (e.g. to visit the App Store from a link on the full screen view).
  • – adViewWillDismissScreen:

    Callback sent just before dismissing the full screen view.
  • – adViewDidDismissScreen:

    Callback sent just after dismissing the full screen view. Use this opportunity to restart anything you may have stopped as part of adViewWillPresentScreen: callback.
  • – adViewWillLeaveApplication:

    Callback sent just before the application goes into the background because the user clicked on a link in the ad that will launch another application (such as the App Store). The normal UIApplicationDelegate methods like applicationDidEnterBackground: will immediately be called after this.
